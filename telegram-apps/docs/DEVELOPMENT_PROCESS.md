# Процесс разработки

Мы придерживаемся следующих принципов профессиональной разработки:

1.  **Жизненный цикл:**
    *   **Design First:** Значимые изменения обсуждаются и фиксируются перед реализацией.
    *   **TDD (Test-Driven):** Код пишется под тесты. Любой баг закрывается добавлением регрессионного теста.
    *   **Continuous Documentation:** Любое изменение функционала отражается в `CHANGELOG.md` и документации проекта.
2.  **Тестирование:**
    *   API-тесты (`pytest` + `requests`) для каждого эндпоинта.
    *   E2E-тесты (`Playwright`) для критических сценариев (загрузка файлов, сохранение промптов).
3.  **Контроль версий (Git):**
    *   Работа ведется в ветках `feature/*` или `fix/*`.
    *   Атомарные коммиты с осмысленными сообщениями.
    *   Все изменения пушатся в удаленный репозиторий.
4.  **Развертывание:**
    *   Тестовая среда (Staging) идентична Production.
    *   Никаких изменений "на лету" без прогона тестов.
