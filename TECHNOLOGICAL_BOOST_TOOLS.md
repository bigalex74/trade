# ТЕХНОЛОГИЧЕСКИЕ ИНСТРУМЕНТЫ ДЛЯ БУСТА (v3.2)

Этот документ содержит рекомендации по обновлению стека технологий для повышения скорости и масштабируемости системы.

## 1. Polars (Lightning-fast DataFrames)
**Замена:** Pandas.
**Зачем:** В 5-10 раз быстрее обрабатывает и рассчитывает технические индикаторы за счет многопоточности на Rust.
**Эффект:** Пайплайны данных обновляются за доли секунды.

## 2. Redis (Hot Data Cache)
**Назначение:** Хранилище в оперативной памяти (In-memory).
**Зачем:** Хранение текущих цен и снимков рынка для мгновенного доступа трейдеров (микросекунды против миллисекунд у PostgreSQL).
**Эффект:** Снижение нагрузки на основную БД и ускорение принятия решений.

## 3. ZeroMQ (Event-Driven Engine)
**Назначение:** Сверхбыстрая шина сообщений.
**Зачем:** Уход от запуска по расписанию (cron). Трейдеры "просыпаются" мгновенно по сигналу от пайплайна данных.
**Эффект:** Нулевая задержка (latency) между приходом данных и реакцией ИИ.

## 4. Semantic Router
**Назначение:** Умная маршрутизация запросов.
**Зачем:** Локальная классификация сложности запроса. Простые задачи уходят в Ollama, сложные — в Gemini Pro.
**Эффект:** Экономия токенов и ускорение ответа на стандартных рыночных ситуациях.
