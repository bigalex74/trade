# Правила работы с n8n в этом проекте (MCP Edition)

## ОСНОВНЫЕ ПРИНЦИПЫ:
1. **Docker-first**: Все операции с n8n проводятся через `docker compose` в папке `/home/user/n8n-docker`.
2. **База данных**: n8n использует PostgreSQL 16 (`db` контейнер). Для проверки статусов и данных использовать `psql`.
3. **Безопасность**: Никогда не выводить API ключи в лог. Все учетные данные в `n8n-docker/.env`.
4. **Прокси**: Для работы с внешними API (Telegram и др.) n8n использует `127.0.0.1:10808` (xray на хосте).

## СТРУКТУРА БД (Критично для работы):
- `document_jobs`: (id, file_name, status, created_at) - основные задачи.
- `document_chunks`: (id, job_id, chunk_text, result_text, status) - части документов.
- `telegram_messages`: логи входящих сообщений.
- `telegram_send_message`: очередь исходящих сообщений.
- `document_log`: системный лог работы воркфлоу.

## ПРАВИЛА ИЗМЕНЕНИЙ:
1. Любое изменение в воркфлоу должно сопровождаться экспортом в `.json` и сохранением в базу знаний LightRAG.
2. При ошибках в n8n - первым делом проверять логи контейнера: `docker logs n8n-docker-n8n-1 --tail 50`.
3. После правок - эмпирическое тестирование.

## MCP ИНТЕГРАЦИЯ:
- n8n доступен как MCP сервер по адресу `http://localhost:5678/rest/mcp/sse`.
- Для управления n8n через CLI использовать `n8n-mcp` (если настроен ключ) или прямые команды `docker exec`.
