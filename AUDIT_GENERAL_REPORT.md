# ОБЩИЙ ОТЧЕТ ПО АУДИТУ TRADE LEAGUE (v3.2)

## 1. Резюме архитектуры
Система представляет собой многоагентную среду для алгоритмической торговли, разделенную на два дивизиона: MOEX (акции РФ) и Crypto. Основной стек: Python, PostgreSQL, Gemini AI, Ollama.

## 2. Ключевые риски
- **Затраты:** Неконтролируемый расход токенов в бэктестере и ежедневной эволюции.
- **Инфраструктура:** Зависимость от Linux-путей (`/home/user`) в Windows-окружении.
- **Стабильность:** Отсутствие асинхронности (блокирующие вызовы API ИИ останавливают всю систему).
- **Исполнение:** Упрощенная логика матчинга ордеров без учета проскальзывания и ликвидности.

## 3. Рекомендации
- Переход на асинхронные запросы (httpx/aiohttp).
- Внедрение динамического сжатия контекста в промптах.
- Использование systemd или docker-compose для управления процессами.

*Детальные отчеты по каждому скрипту находятся в папке `audit_reports/`.*
