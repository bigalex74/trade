# Исправление ложной ошибки AI fallback

Дата: 2026-04-23

Ветка изменения: `TRADE-005`

Версия проекта после изменения: `0.2.1`

## Симптом

В Telegram появлялось сообщение:

```text
Scalper_Kesha: Ошибка! Все AI модели недоступны
```

При проверке логов оказалось, что это сообщение было неточным. Не все модели были недоступны.

## Что показали логи

По `Scalper_Kesha` были видны три разные причины:

- prompt оставался больше лимита `8000` символов даже после отключения RAG-блока;
- часть моделей пропускалась локальным `health guard` из-за cooldown;
- timeout одной модели останавливал fallback-цепочку, поэтому следующие модели не пробовались.

Это приводило к ложному Telegram-выводу, будто недоступны все модели.

## Исправления

### Timeout fallback

В `gemini_cli_runner.py` добавлен per-attempt timeout.

Для трейдеров в `run_ai_trader_once.sh` установлено:

```text
GEMINI_ATTEMPT_TIMEOUT_SECONDS_TRADER=35
```

Общий лимит запуска остался:

```text
GEMINI_TIMEOUT_SECONDS=120
```

Теперь timeout одной модели не обрывает весь fallback. Runner идет к следующей модели, пока не закончится общий лимит или fallback-budget.

### Health guard

В `ai_cost_guard.py` исправлен подсчет recent failures:

- раньше recent failures считались по модели без фильтра по категории;
- теперь failures считаются внутри категории, например только `trader`.

Это исключает ситуацию, когда ошибки отчета, watchdog или другого AI-скрипта ухудшают здоровье модели для биржевого трейдера.

Также успешный вызов модели очищает ее cooldown в этой категории.

### Prompt budget

В `ai_paper_trader.py` добавлен дополнительный guard:

- сначала ужимается или отключается RAG;
- если prompt все еще больше лимита, market features сокращаются до приоритетных тикеров;
- минимальное число тикеров задается через:

```text
AI_TRADER_MIN_SYMBOLS=5
```

Для Meta Oracle дополнительно уменьшены дефолты:

```text
AI_META_ORACLE_LEAGUE_TRADES_LIMIT=12
AI_META_ORACLE_RATINGS_LIMIT=6
```

Это не меняет стратегию принятия решений, а предотвращает полный отказ AI-вызова из-за превышения prompt-budget.

### Telegram

Сообщение больше не говорит `Все AI модели недоступны`.

Теперь трейдер отправляет более точную причину:

- запрос превысил лимит;
- сработал часовой лимит ИИ-запросов;
- модель не ответила за отведенное время;
- модель временно на паузе после предыдущей ошибки;
- модель вернула лимит или перегрузку;
- другая конкретная ошибка из `trading.ai_call_log`.

## Проверка

Добавлен тест:

```bash
tests/run_ai_fallback_smoke.sh
```

Он проверяет сценарий:

1. первая модель зависает;
2. runner ловит timeout;
3. runner продолжает fallback;
4. вторая модель успешно возвращает JSON.

Также проверен dry-run `Scalper_Kesha`:

- RAG был отключен из-за budget;
- market features автоматически сокращены с `8` до `7` тикеров;
- итоговый prompt: `7787` символов;
- AI-вызов успешен;
- заявка не записана, потому что запуск был dry-run.

## Операционный вывод

Сообщение в Telegram было следствием некорректной диагностики и слишком жесткой fallback-логики, а не доказательством фактической недоступности всех Gemini-моделей.

После исправления трейдер должен либо получить AI-решение через fallback, либо отправить точную причину отказа.
