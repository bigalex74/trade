# Правило маршрутизации баз знаний

Дата: 2026-04-23

## Правило

Есть две разные базы знаний, и их нельзя смешивать:

- `https://algo.bigalexn8n.ru` - база знаний трейдеров.
- `https://kb.bigalexn8n.ru` - общая база знаний для технического и операционного контекста.

## Что писать в ALGO KB

В `algo.bigalexn8n.ru` попадает только информация, которая помогает трейдерам принимать торговые решения:

- рыночные обзоры;
- торговые гипотезы;
- стратегические отчеты;
- риск-контекст;
- результаты backtest/holdout, если они влияют на торговые решения;
- качественные записи о поведении стратегий и сигналов.

Трейдерам не нужны записи о git, переносе папок, cron housekeeping, Codex-операциях, структуре репозитория и прочей инфраструктуре, если это не влияет напрямую на торговое решение.

## Что писать в General KB

В `kb.bigalexn8n.ru` попадает все, что нужно для восстановления контекста задач и будущей разработки:

- что менялось в git и репозитории;
- где теперь живет проект;
- какие cron-задачи переключались;
- какие проверки выполнялись;
- инциденты инфраструктуры;
- планы работ;
- операционные правила для Codex.

## Операционное правило для следующих задач

Если запись отвечает на вопрос "что должны знать трейдеры для торговли?", ее можно писать в ALGO KB.

Если запись отвечает на вопрос "что должен помнить Codex/разработчик/оператор?", ее нужно писать в General KB.
