# TRADE-011: общий бюджет AI-вызова

Дата: 2026-04-23

Версия: 0.4.4

## Что изменено

Общий бюджет Gemini-вызова увеличен с `120` до `200` секунд.

Измененные дефолты:

```bash
GEMINI_TIMEOUT_SECONDS=200
GEMINI_ATTEMPT_TIMEOUT_SECONDS_TRADER=60
GEMINI_CONCURRENCY=1
```

## Почему

После увеличения timeout одной попытки до `60` секунд fallback-цепочке стало не хватать общего бюджета `120` секунд. Если первые модели зависали или отвечали медленно, поздние fallback-модели получали остаток общего лимита, например `17` секунд, и не успевали дать решение.

Новый общий бюджет `200` секунд дает нескольким моделям шанс отработать полноценные `60` секунд, не меняя качество prompt и не расширяя входные данные.

## Где применено

- `run_ai_trader_once.sh`
- `run_ai_crypto_trader_once.sh`
- `run_ai_job_dispatcher.sh`
- `gemini_cli_runner.py`
- `tests/run_trader_runtime_config_smoke.sh`

## Ожидаемый эффект

- Меньше ситуаций, когда поздняя fallback-модель получает слишком короткий остаточный timeout.
- Меньше отказов вида `ИИ-решение не получено` из-за исчерпания общего бюджета.
- Больше времени на цепочку старших моделей без изменения лимита одной попытки.

## Риски и контроль

Риск: одна волна трейдеров может выполняться дольше, если несколько моделей подряд зависают до timeout.

Контроль:

- `GEMINI_CONCURRENCY=1` сохраняет защиту от параллельного перегруза Gemini CLI.
- `GEMINI_ATTEMPT_TIMEOUT_SECONDS_TRADER=60` не дает одной модели занимать весь общий бюджет.
- Логи `trading.ai_call_log` должны показать меньше коротких timeout у поздних fallback-моделей.
