# ПЛАН РЕАЛИЗАЦИИ ВЕКТОРНОЙ ПАМЯТИ (QDRANT RAG)

**Статус:** Ожидает подтверждения (Pending Approval)

## 1. Инфраструктурный слой
- **Инструмент:** Qdrant DB (в Docker-контейнере).
- **Эмбеддинги:** Библиотека `fastembed` (BGE-Small-EN-v1.5) — работает локально на CPU, не потребляет платные API-квоты.
- **Интеграция:** Создание `docker-compose.qdrant.yml` для поднятия базы на порту `6333`.

## 2. Слой Индексации (Запись Опыта)
- **Скрипт:** `ai_memory_indexer.py` (запускается по cron в 01:00).
- **Логика:** 
  1. Ищет в `trading.journal` пары сделок (BUY -> SELL или SHORT -> COVER).
  2. Вычисляет итоговый PnL сделки.
  3. Если сделка успешная (PnL > 1%), берет `indicators_snapshot` из момента открытия.
  4. Формирует текстовое описание рыночного контекста и конвертирует в вектор.
  5. Сохраняет в коллекцию `successful_setups`.

## 3. Слой Извлечения (Чтение Опыта трейдерами)
- **Изменения в `ai_crypto_trader.py`:**
  - Добавление функции `query_vector_memory(current_snapshot)`.
  - Перед отправкой промпта в Gemini, скрипт векторизует текущее состояние целевого актива (например, BTC) и ищет ТОП-3 ближайших совпадения в Qdrant.

## 4. Слой Промптинга (Prompt Injection)
В финальный промпт ИИ-агента будет добавлен блок `HISTORICAL EXPERIENCE`.
**Пример:**
> "Векторная память нашла 2 похожих исторических сетапа с точностью 89%. 
> В 100% случаев при таких индикаторах покупка приносила в среднем +3.2% прибыли. 
> Учитывай этот паттерн при формировании уверенности (confidence)."

## 5. Ожидаемый результат
ИИ-агенты начнут накапливать "интуицию". Вместо голого математического анализа текущей свечи, они смогут опираться на эмпирический опыт всей Лиги Трейдеров.
