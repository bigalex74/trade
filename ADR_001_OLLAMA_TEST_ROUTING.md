# ADR 001: Внедрение тестового контура через Ollama

## Статус
Предложено (Proposed)

## Контекст
Система перевода новелл использует платные API (Polza.ai, Neuro AI). Для проведения E2E тестирования и отладки промптов требуется бесплатный контур, который не будет расходовать баланс и не затронет основные (Production) задачи.

## Решение
1. **Флаг в БД**: В таблицу `document_jobs` добавлена колонка `is_test` (BOOLEAN).
2. **Динамическая маршрутизация**: В воркфлоу `[Перевод] Перевод чанка` (ID: `GPARI8V4RBSPL1h39_kHW`) внедряется узел `Switch` после получения метаданных задачи.
3. **Провайдеры**:
    - `is_test = true` -> Локальная Ollama (модель: llama3.2:3b).
    - `is_test = false` -> Платные API (GPT-4o/Claude).
4. **Безопасность**: Прямое соединение n8n -> Ollama (localhost:11434) через host-network.

## Последствия
- **Плюсы**: Бесплатные E2E тесты, изоляция ошибок, возможность отладки без риска для бюджета.
- **Минусы**: Модель Llama 3.2 3b может давать менее качественный перевод, что допустимо для технических тестов.
