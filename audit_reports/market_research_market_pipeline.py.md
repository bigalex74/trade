# АНАЛИЗ СКРИПТА: market_research_market_pipeline.py

## 1. Архитектура и Логика
- **Назначение:** Сбор данных с MOEX и расчет технических индикаторов.
- **Стек:** `requests` (API ISS MOEX), `pandas`, `pandas_ta`.
- **Тип:** ETL-пайплайн (Extract, Transform, Load).
- **Разрыв:** Синхронный последовательный опрос всех тикеров. При большом количестве бумаг процесс занимает много времени и может не успеть обновить данные к началу торгового часа.

## 2. Отказоустойчивость
- **Сеть:** Есть базовая обработка ошибок запросов, но при 429 ошибке (Rate Limit) от MOEX скрипт может просто прекратить работу.
- **БД:** Массовая вставка данных (`execute_batch`) реализована хорошо для производительности.

## 3. Безопасность
- **SQLi:** Используется безопасная вставка.
- **Хардкод:** Опять же, пути к конфигам и логам.
- **API:** Используется публичное API MOEX, ключи не требуются, что упрощает безопасность.

## 4. Алгоритмические недостатки
- **Избыточность:** Рассчитывается более 50 индикаторов, из которых трейдеры используют максимум 5-10. Это пустая трата ресурсов CPU и места в БД.
- **Качество данных:** Нет проверки на аномальные скачки (Outliers). Если API вернет ошибочную цену, индикаторы (особенно RSI и ATR) будут искажены на несколько дней вперед.

## 5. Критические изменения и Тюнинг
- **Тюнинг:** Сократить список рассчитываемых индикаторов до реально необходимых.
- **Изменение:** Внедрить многопоточность (`threading` или `asyncio`) для опроса API MOEX.
- **Обновление:** Добавить валидацию данных (Price sanity check) перед записью в БД.
