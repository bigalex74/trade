# АНАЛИЗ СКРИПТА: ai_backtester.py

## 1. Архитектура и Логика
- **Назначение:** Историческое тестирование стратегий ИИ-агентов.
- **Разрыв:** Самый большой потребитель токенов. Делает вызов ИИ для каждой исторической свечи.
- **Логика:** Упрощенное исполнение (всегда по 1 акции/лоту), что не позволяет тестировать стратегии управления капиталом (Money Management).

## 2. Отказоустойчивость
- **Таймауты:** Использование `timeout=120` для `subprocess` может быть недостаточно для сложных промптов Gemini 1.5 Pro.
- **Данные:** Если в БД есть пропуски в истории, бэктестер "перепрыгивает" их, что искажает доходность.

## 3. Безопасность
- **SQLi:** Чисто.
- **Ресурсы:** Скрипт может "съесть" весь бюджет на API за один запуск, если не ограничить период тестирования.

## 4. Алгоритмические недостатки
- **Look-ahead Bias:** Промпт содержит данные текущего дня. ИИ видит High/Low дня и может "предсказывать" будущее, выдавая нереально высокую доходность (Grail effect).
- **Slippage:** Нулевое проскальзывание и отсутствие комиссий делают результаты теста слишком оптимистичными.

## 5. Критические изменения и Тюнинг
- **Тюнинг (ОБЯЗАТЕЛЬНО):** Внедрить пакетную обработку (Batching). Вместо 1 запроса на 1 день, слать данные за неделю и просить ИИ выдать решения по всем дням сразу. Это сократит расходы в 5-7 раз.
- **Изменение:** Добавить учет комиссий биржи и проскальзывания.
- **Обновление:** Запретить ИИ видеть данные "будущего" (свечу закрытия текущего периода).
